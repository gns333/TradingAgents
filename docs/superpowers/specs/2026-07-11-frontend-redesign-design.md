# TradingAgents 前端重构设计

- **日期**：2026-07-11
- **范围**：`tradingagents/web/static/` 下的 `index.html` / `workbench.css` / `workbench.js`
- **目标**：把当前朴素的浅绿商务风前端，重构为专业深色金融终端，提升信息层次、可读性与使用体验，同时保持所有现有功能与 API 契约不变。
- **约束**：保持原生 JS + 手写 CSS，零构建，FastAPI 直供静态文件。不引入框架，不改后端。

## 1. 设计定位

- **风格**：Dark Mode (OLED) 专业金融交易终端。深色为主，右上角一键切换浅色，`localStorage` 记忆偏好。
- **调性依据**：ui-ux-pro-max 数据库 fintech/trading dashboard 推荐 —— Dark OLED 风格、Inter 字体、数据密集布局、WCAG AAA 对比。
- **Signature（记忆点）**：**Agent 协作流水线可视化** —— 7 个 agent 连成一条带连接线与实时脉冲的流水线，终点是醒目的 Buy/Sell/Hold 决策徽章。这体现产品最独特的"多 Agent 协作决策"，而非套模板的大数字 Hero。boldness 只花在这一处，其余保持克制。

## 2. 设计 Token

### 2.1 配色（深色为默认）

| Token | 深色值 | 浅色值 | 用途 |
|---|---|---|---|
| `--bg` | `#0D1117` | `#F5F7FA` | 页面底色 |
| `--surface` | `#161B22` | `#FFFFFF` | 面板背景 |
| `--surface-2` | `#1C2330` | `#F0F3F7` | 嵌套/悬浮层 |
| `--line` | `#2A3441` | `#DDE3EB` | 分隔线/边框 |
| `--line-soft` | `#20262F` | `#EDF1F5` | 弱分隔线 |
| `--text` | `#E6EDF3` | `#1A2029` | 主文字 |
| `--muted` | `#8B98A9` | `#5C6B7E` | 次要文字 |
| `--faint` | `#5C6675` | `#8B97A7` | 更弱文字 |
| `--accent` | `#F5A623` | `#B87400` | 品牌强调（导航激活/主按钮/聚焦/流水线高亮） |
| `--accent-soft` | `rgba(245,166,35,.14)` | `rgba(184,116,0,.10)` | 强调弱背景 |
| `--up` | `#F6465D` | `#E0384E` | 涨 / 买入 / 看多（红） |
| `--up-soft` | `rgba(246,70,93,.14)` | — | 涨弱背景 |
| `--down` | `#2EBD85` | `#1A9E6C` | 跌 / 卖出 / 看空（绿） |
| `--down-soft` | `rgba(46,189,133,.14)` | — | 跌弱背景 |
| `--warning` | `#F0A020` | `#9A5B00` | 警示 |

**配色关键决策**：数据库所有 fintech 方案都用绿色做品牌 accent，但本项目要红涨绿跌，绿色已语义化为"跌"，不能兼作品牌色（否则导航激活态与"下跌"同色，语义冲突）。因此把红/绿完全让给涨跌语义，品牌强调色另选**琥珀橙 `#F5A623`** —— 与红绿零冲突，且是 Bloomberg 式金融终端的经典调性。

- 所有功能色（涨跌、决策、状态）**不只用颜色**，同时配箭头/图标/文字（满足 `color-not-only`，色盲友好）。
- 涨跌/决策映射：涨·买入·看多 = 红；跌·卖出·看空 = 绿；持有·中性 = 灰。

### 2.2 字体

- 引入 Google Fonts：`Inter`（界面文字）+ `JetBrains Mono`（数字/价格/代码/数据表）。
- 字体栈带系统回退，`font-display: swap`。
- **数字一律用 JetBrains Mono + `tabular-nums`**：价格、涨跌幅、指标值、每日次数、时间戳、事件计数。等宽数字是金融终端的灵魂，防止跳动、便于纵向对齐。
- 字号阶：`12 / 13 / 14 / 16 / 18 / 22 / 28`。权重：标题 600–700，正文 400，标签 500。

### 2.3 间距与形状

- 间距节奏：`4 / 8 / 12 / 16 / 24 / 32`（density 8/10 数据密集档，偏紧凑）。
- 圆角：`--radius: 6px`（终端感收紧），徽章/pill `999px`。
- 层次靠**边框 + 微妙背景差**营造，弱化阴影（深色下大阴影无意义）。

## 3. 整体布局

保留侧边栏导航 + 三视图结构（个股分析 / 报告中心 / 后台管理）。顶栏重构：品牌 · **股票信息条** · 运行状态 pill · 主题切换按钮 · 管理员入口。

### 3.1 个股分析（核心页）三栏终端布局

```
┌─ 顶栏：TA · 002558.SZ 巨人网络 ¥29.56 ▲+2.3% · 运行状态 · 主题切换 · 管理员 ─┐
├──────────┬─────────────────────────────────┬──────────────────┤
│ 左 320px  │        中栏（报告主舞台，最宽）        │  右 300px         │
│ 分析参数   │  当前报告（Tab 切换各分析师）          │  Agent 团队流水线  │
│ (sticky)  │  决策徽章置顶 + 结构化 Markdown       │  ── 过程事件流 ──  │
└──────────┴─────────────────────────────────┴──────────────────┘
```

- 把报告从原来的右下角小格子提升为**中央主舞台**，长报告阅读不再憋屈。
- Agent 团队 + 过程日志合并进右栏：流水线在上，事件流在下。
- 响应式：≥1200px 三栏；768–1200px 两栏（右栏落到底部）；<768px 单栏堆叠。

### 3.2 报告中心

- 顶部新增**筛选/搜索条**（纯前端过滤已加载列表）：按代码/名称搜索 + 按决策类型筛选（全部/买入/卖出/持有）。
- 左栏历史列表升级为**卡片式**：股票名+代码、交易日期、决策徽章、模块小标签、归档时间。选中项品牌色高亮条。删除按钮 hover 显现。
- 右栏详情顶部加**报告摘要头**（股票信息 + 决策徽章 + 日期 + 参与 Agent），下接 Tab 分栏报告正文（复用分析页同款终端风 Tab 组件）。

### 3.3 后台管理

- 顶部**状态条**：管理员会话状态 + 退出按钮，横贯顶部（不再单独占大面板）。
- 左栏配置表单区：模型配置卡 + 白名单卡，恢复合理间距与字段分组（现状 `gap:2px` 过密），供应商/模型/Key 输入统一终端风。保存反馈用色块状态提示。
- 右栏当前配置总览：模型表（默认徽章 + Key 掩码 + 供应商小标签）、白名单表（状态色点徽章 + 等宽次数），表头 sticky、hover 高亮行、深色重配色。

## 4. 组件重设计要点

- **股票信息条**：选定股票后顶栏显示 `代码 名称 价格 涨跌幅`，红涨绿跌 + 箭头。数据来源沿用现有股票搜索接口返回的 name/code；价格若无实时源则仅显示代码名称，不虚构数据。
- **Agent 流水线**：紧凑行式卡片 + 连接线，三态（进行中脉冲动画 / 已完成 / 待处理）用色点 + 文字区分。当前工作角色品牌色高亮。
- **决策徽章**：Buy/Sell/Hold 醒目色徽章（买入红 / 卖出绿 / 持有灰），置于报告顶部与历史卡片。
- **过程时间线**：紧凑化，工具调用 / Agent 输出 / 报告更新用不同左边框色 + 小图标区分，长文本折叠保留。
- **Markdown 报告**：深色下重配表格/标题/代码块；数据表数字右对齐等宽；`<pre>` 深色底。
- **表单控件**：输入/下拉/复选统一深色描边，聚焦态品牌色高亮 ring。

## 5. 动效（motion 4/10，克制）

- 微交互 150–300ms，`ease-out` 进入。
- 仅两处关键动效：流水线当前节点脉冲、Tab/主题切换的平滑过渡。
- 遵守 `prefers-reduced-motion`：关闭脉冲与过渡。

| 文件 | 改动 |

**不做**：不引入框架/构建、不改后端 API、不改 Python、不做无关重构、不改事件契约。

## 7. 验证策略（最小化）

按用户要求，不跑回归测试套件。仅在实现完成后做**一次**整体验证：用 Playwright 起本地 web 服务，截图确认三视图排版正常、深/浅色切换正常、Markdown 报告渲染正常；若能触发一次分析流，观察流水线实时更新与决策徽章。风险：未做跨浏览器与后端集成回归，但本次改动集中在展示层、不触碰数据契约，风险可控。
