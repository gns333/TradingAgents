# 报告中心卡片操作区修复设计

## 背景与根因

报告中心的历史报告卡片同时把决策徽标和“删除”按钮放在卡片右侧。决策徽标属于标题行，而删除按钮通过绝对定位覆盖在相同区域；卡片启用了 `overflow: hidden`，因此在当前列宽下会出现徽标、删除文字相互覆盖并被右边界截断的问题。线上加载的 CSS 与仓库代码一致，排除缓存和旧版本资源。

## 方案比较

1. **独立操作列（采用）**：卡片使用两列布局，左列承载报告内容和决策徽标，右列固定承载删除按钮。结构清晰，桌面端和窄屏都不会重叠。
2. **继续绝对定位并增加留白**：改动更少，但留白需要同时容纳徽标和删除按钮，容易受文案、字体和屏幕宽度影响再次截断。
3. **把删除操作收入菜单**：空间占用最小，但增加交互步骤和新的菜单状态，超出本次缺陷修复范围。

## 设计

- `.history-item` 改为两列网格：`minmax(0, 1fr)` 内容列加 `auto` 操作列。
- `.history-open` 取消为悬浮删除按钮预留的右侧大内边距，并确保内容列可以收缩。
- `.history-delete` 回到正常网格流，取消绝对定位和位移，使用左边框与内容区分隔，并保持始终可见。
- 决策徽标继续位于标题行右侧；标题和元信息仍按现有规则省略过长文本。
- 小屏下删除按钮继续显示为紧凑的“×”图标，但仍占独立列，不覆盖报告内容。
- 不修改报告数据、接口、删除确认流程或详情区域。

## 验证

- 增加静态样式回归测试，要求历史卡片存在独立两列布局，删除按钮不再绝对定位。
- 运行报告中心静态契约测试，并对 CSS/JavaScript 做最小语法与内容检查。
